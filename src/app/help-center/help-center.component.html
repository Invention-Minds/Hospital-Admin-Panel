New Appointment Form
Form Fields for Appointment Booking :
  • PRN Field to enter the patient's PRN.
  • Patient Name Fields for entering the patient's first and last name.

Appointment Module
Confirmed Appointments
This section lists all appointments that have been successfully confirmed by the admin or sub admin. It helps track and manage upcoming visits in real time.
Heres what you can do :
  • Reschedule or Cancel : Modify appointment time or cancel if required
  • Check-In : Once the patient arrives, click Check-In to notify the doctor and update the TV/queue display
Check-In Timing Rule :
  • Allowed only 30 minutes before or after the scheduled time
  • If not checked in within this window, the system auto-cancels the appointment
  • Notifications are sent to both the patient and doctor on auto-cancel
This ensures smoother queue management and timely consultations.

Appointment Module
Transferred Appointments
This section displays appointments that were transferred by the doctor to another doctor or department.
Doctor's Action :
  • The doctor initiates a transfer when a case needs referral or another departments opinion. After transfer, the original appointment must be either Completed or Canceled. This decision is taken based on the doctor's notes or instructions at the time of transfer
  • After completing or canceling, sub-admin must manually create a new walk-in appointment under the referred doctor to continue the patients journey.
  • This ensures proper tracking of both the original consultation status and the new follow-up with the transferred doctor.

Doctor Module
Doctor Availability
This section helps the admin manage and view a doctors appointment slots efficiently.
Slot Color Coding :
  • Green Slot : Available slots that patients can book for appointments.
  • Grey Slot : Unavailable slots, indicating times when the doctor is not available.
  • Red Slot : Booked slots that are already scheduled.
  • Blue Slot : Completed slots, indicating that the consultation has been completed.
  • Yellow Slot : Blocked slots that cannot be booked.
  • Violet Slot : Extra slots added beyond the doctors regular OPD hours.
Date Navigation :
Admins can use the "< Prev" and "Next >" buttons to navigate across different dates and view availability.
Search Doctor & Date Filter :
Admins can search for a specific doctor or select a particular date to view their slot availability.
Unavailable Slots :
If a doctor is unavailable due to emergencies or has to leave the OPD early, slots can be marked as unavailable.
Add Slots :
Admins can add extra slots before or after the doctors standard OPD time if required.

Doctor Module
Appointment-Based Doctor - Visiting Consultant (Surgeon)
This feature is designed for doctors who work on an appointment-only basis without fixed OPD schedules, such as visiting consultants or surgeons.
Flexible Slot Addition:
  • Visiting consultants typically do not have predefined working hours.
  • Admins can manually add slots as needed based on the consultants availability or surgical schedule.
Doctor Type Selection :
While adding a doctor to the system, the admin must select the appropriate doctor type:
  • Retain Doctor :
    Assigned with predefined availability slots that follow a regular OPD schedule.
  • Visiting Consultant (VC) :
    No fixed slots. Slots must be added manually whenever the consultant is available.
Doctor Availability View :
  • For Visiting Consultants, only the booked slots appear in the Doctor Availability section.
  • Unbooked or future slots will not be displayed unless manually created.
This ensures that visiting doctors availability is managed flexibly, allowing hospitals to accommodate specialists without affecting standard scheduling workflows.

Estimation Module
Estimation Creation
This section allows the Estimator to create a detailed cost estimation for procedures or surgeries, capturing all necessary information for clarity and accuracy.
Key Sections & Functionalities :
    a. Patient Information
    • Enter the patient's UHID, Name, Age and Gender to link the estimation with the correct patient profile.
    b. Doctor Information
    • Select the Doctors Name responsible for the procedure or consultation.
    c. Estimation Details
    • Choose the Estimation Type
    • Enter the EST Name and Surgery Level
    • Provide the Doctors Preferred Date for the surgery
    d. Stay Information
    • Input the total number of stay days
    • Specify if ICU or Ward requirements are applicable
    e. Room Information
    • View and select from the available room types
    • Enter the room-wise charges for cost calculation
    f. Inclusions and Exclusions
    • Select items and services that should be included in the estimation
    • Unselected items will be automatically treated as exclusions
    • Helps define the exact scope of services provided
    g. Implants, Procedures, and Instruments
    • Add multiple items under each category
    • Assign cost values individually
    • If marked as included, their cost will not be added to the final estimation
    h. Estimation Summary
    • Displays :
      • Estimation Date
      • Total Estimated Cost
      • Automated Cost Calculation (based on room charges and excluded items)
      • Internal Remarks (for internal staff use)
      • External Remarks (shared with the patient)
    i. Signatures
    • Mandatory digital signatures from:
      • Staff
      • Approver
      • Patient or Attender
    j. Rejection Reason
    • If an estimation is rejected, the approver must provide a valid reason for documentation and clarity.
This feature ensures transparency, accuracy and accountability in cost planning for patients and supports smooth pre-admission processes.

Individual Doctor Module
Checked - In Consultations
This section displays the list of patients who have checked in for today's appointments. Doctors can take actions directly from this view to manage their consultations efficiently.
Table Types :
The doctors screen is divided into three views :
  1. Checked-In Consultations - Patients currently waiting or being seen
  2. Todays Consultations - All scheduled appointments for the day
  3. Future Consultations - Upcoming patient appointments
  4. Past Consultations - Past patient appointments
Actions Available (Checked-In Tab) :
  • Start :
    Begins the consultation. On the TV screen, the patient status updates to "Patient-In".
  • Finish :
    Ends the consultation. On the TV screen, the patient's name is removed from the display.
  • Postpone :
    Postpones the appointment. The status is updated to "Pending" on the TV display.
  • P. Counselling (Patient Counselling) :
    Creates a new estimation request with details like:
    • Surgery Name
    • Estimation Type (SM, MM, Maternity)
    • Preferred Date
    • Stay Duration
    • Surgery Level
    • Type (Immediate or Planned)
    • Doctors Remarks
  • Transfer :
    Sends a request to the manager to transfer the appointment to another doctor or department.
  • CC (Consultation Referral) :
    Sends a referral request to the sub-admin, suggesting the patient consult another doctor for additional opinion or treatment.
Visual Indicators :
  • Color-Coded Buttons :
    Each action button is uniquely colored for easy identification.
  • Timer Display :
    A live timer tracks how long each consultation is ongoing.
    • If the consultation exceeds 10 minutes from the slot duration of that doctor, the row starts blinking in red to indicate delay.
This module helps doctors manage patient flow smoothly, communicate with other departments and ensure timely consultations.

    Individual Doctor Module
    Close OPD
    This feature allows doctors to close their OPD slots for checked-in consultation for the day in case of emergencies or sudden unavailability.
    Key Functionalities :
  • Select Slots to Close :
    Doctors can choose specific available appointment slots that need to be closed for the day.
  • Slot Selection :
    A list of available time slots is shown. Doctors can :
    • Select individual slots
    • Or use Select All to close all remaining slots at once
  • Submit Action :
    Once submitted, the selected slots are closed immediately, preventing any further bookings or check-ins for those times.
    • This feature ensures smooth handling of unexpected changes in the doctors schedule, while keeping patients and staff informed.
    Individual Doctor Module
    End Consultation
    This feature is used by doctors to end their OPD for the day after all scheduled consultations are completed.
    Key Functionalities :
  • Remove from Queue :
    Once clicked, the doctor is removed from the live queue, and their name will no longer appear on the display screens or patient list for the day.
    • This helps ensure the system reflects real-time availability, preventing any further patient flow to the doctor once the days consultations are finished.
    TV Control Module
    This module manages the real-time display of doctor appointments, patient queue flow, and promotional content on hospital TV screens.
      1. Queue Management
      • Displaysdoctor-wise appointments in each TV channel.
      • Shows real-time statuses such as:
        • Patient In
        • Next
        • Appointment Time
        • Pending
      • Includes actual and predicted wait times for better queue visibility.
      2.Advertisements
      • Allows the display of text and image-based advertisements on the TV screen.
      • Ads are managed and scheduled through the queue management system to run between appointment updates.
      3.Doctor Assignment in Channel
      • An automated system assigns doctors to specific TV channels based on their room numbers.
      • Ensures that patients see only the relevant doctor queues per department or area.
    This module helps in streamlining patient flow visibility, while also offering space for promotions and information display across hospital premises.
    Analytics
    Previous Analytics Data
    1. OPD Request via -This chart displays the distribution of OPD requests over the last seven days based on booking mode - Online, Call or Walk-In. It highlights patient preferences, helping promote online booking for better convenience and reduced call volume.
    2. OPD Sessions -This chart compares OPD visits during Morning and Evening sessions. It provides insights into patient preferences, enabling efficient staff allocation and resource planning.
    3. OPD Overview - This chart tracks OPD appointment statuses, including Total Appointments, Confirmed, Completed, Canceled and Check-Ins. It helps monitor operational performance and identify trends in patient flow.
    4. OPD Type - This chart categorizes OPD visits into Paid, Follow-up, Camp, and MHC. It offers insights into appointment types, assisting in resource allocation and service optimization.
    5. OPD Estimation Types -This chart displays the distribution of estimation types from OPD visits, categorized as MM, SM, or Maternity. It helps in understanding predominant estimation types for better resource planning.
    6. Estimation Status Overview - This chart tracks the status of OPD estimations, including Pending, Submitted, Approved, Confirmed, Completed, Cancelled and Overdue. It helps monitor estimation progress and identify bottlenecks.
    7. Average Waiting Time -This chart shows the number of patients who waited over 40 minutes for a doctor consultation. It helps identify delays and improve operational efficiency.
    8. Gender Overview -This chart displays the gender distribution of patients over the past seven days, visualizing the male-to-female ratio for demographic insights.
    9. MHC Waiting Time -This chart tracks the number of patients who waited more than 30 minutes for their respective MHC (Master Health Check) packages in the last seven days. The packages include various health check-ups such as Diabetic Care, Senior Citizen Health Checks, and Executive Health Checks.
    10. MHC Overview - This pie chart provides an overview of MHC package bookings over the last seven days. It highlights the number of people who booked each package, including options like Integrated Diabetic Care, Annual Master Diabetes Care, Senior Citizen Health Checks, and Executive Health Checks.
    View More
    Each chart includes a "View More" button, which, when clicked, provides an in-depth, enlarged view of one month data for better visibility and detailed analysis.
    Details
    Each chart features a "Details" button that directs users to the report section. Here, admins can download reports in Excel format for further analysis.
    Filters
    The dashboard includes filters that allow users to refine data based on department, doctors, date, and package. These filters offer flexibility in viewing specific information, making analysis more precise.

    • Number of Consultations and Radiology Services included in the package

    • Lab:

      Opens a form displaying:

    • Complete :
      Used to mark the MHC package as completed.
      • If any consultation or radiology appointments are not checked in, they will be auto-cancelled with acknowledgement from the MHC Coordinator.

    MHC Coordinator Module
    Report Summary
    The Report Summary section provides a consolidated view of each patient's Turnaround Time (TAT) across various services included in their MHC package.
    Key Functionalities :
    • Patient and Package Details :
    • Displays :
      • Date
      • PRN
      • Patient Name
      • Age
      • Gender
      • Package Name
    • TAT Tracking :
      • TAT (Laboratory) : Time taken to complete lab-related services
      • TAT (Radiology) : Time recorded for completing radiology procedures
      • TAT (Consultations) : Duration for completing all doctor consultations
      • Grand Total : Cumulative time spent across all services for the patient
    • Filters and Export Options :
      Users can filter by date using the calendar and export or print the summary for documentation or review purposes.
    This section helps coordinators and admin teams monitor the efficiency of service delivery and improve the overall experience for MHC patients.
